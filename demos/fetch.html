Calls https://jsonplaceholder.typicode.com/users with the Fetch API. Demonstrates the 3-state pattern (loading, success, error) and dynamic rendering of data in the DOM.

How It Works

When clicking the "Load Users" button, a loading spinner is shown. Then fetch(url) is called, returning a Promise. .then(res => res.json()) parses the response and .then(data => renderUsers(data)) displays the data. If there is an error (network, HTTP not-ok, etc.), it is caught with .catch() and an error message is shown.

function loadUsers() {
  statusEl.textContent = 'Loading...';
  dataEl.innerHTML = '';

  fetch('https://jsonplaceholder.typicode.com/users')
    .then(function (res) {
      if (!res.ok) throw new Error('HTTP ' + res.status);
      return res.json();
    })
    .then(function (users) {
      statusEl.textContent = users.length + ' users loaded';
      users.forEach(function (u) {
        var card = document.createElement('div');
        card.className = 'user-card';
        card.innerHTML = '<strong>' + u.name + '</strong><br>' +
          '✉ ' + u.email + '<br>' +
          '📍 ' + u.address.city;
        dataEl.appendChild(card);
      });
    })
    .catch(function (err) {
      statusEl.textContent = 'Error: ' + err.message;
    });
}

res.ok is checked to detect HTTP errors (4xx/5xx). Each user is rendered as a card with name, email and city, inserted with appendChild.

JavaScript APIs Used

  • fetch() — Modern API for HTTP requests, returns a Promise
  • Response.json() — Parse the response as JSON
  • Promise.then() / .catch() — Async chaining for data flow and error handling
  • document.createElement() — Dynamic creation of DOM elements for each user
  • Element.innerHTML / appendChild() — Insert rendered content into the DOM
