Records audio from the microphone via MediaRecorder, displays a live frequency visualization, and lets you play back or download the recording as a .webm file.

How It Works

Recording: Gets mic access with getUserMedia({ audio: true }), passes the stream to both a MediaRecorder (which stores chunks in an array) and an AnalyserNode for visualization. A timer updates every 100 ms. On stop, the chunks are merged into a Blob and an object URL is created with URL.createObjectURL().

stream = await navigator.mediaDevices.getUserMedia({ audio: true });
mediaRecorder = new MediaRecorder(stream);
mediaRecorder.ondataavailable = function (e) {
  if (e.data.size > 0) chunks.push(e.data);
};
mediaRecorder.onstop = function () {
  var blob = new Blob(chunks, { type: 'audio/webm' });
  var url = URL.createObjectURL(blob);
  /* enable Play button and Download link */
};

Visualization: The same mic stream connects to AudioContext.createMediaStreamSource() → AnalyserNode (fftSize: 128). getByteFrequencyData() fills a Uint8Array with frequency amplitudes, mapped to colored bar heights via hsl().

Playback & Download: The object URL is set as the src of a new Audio() element for playback, and as the href of a download link with download="recording.webm".

JavaScript APIs Used

  • MediaRecorder — Records media streams, fires ondataavailable with chunks and onstop when done
  • MediaDevices.getUserMedia() — Access to the user's microphone
  • AudioContext.createMediaStreamSource() — Connects the mic stream to the Web Audio graph
  • AnalyserNode — Real-time frequency analysis with FFT (getByteFrequencyData)
  • Blob + URL.createObjectURL() — Bundles recorded chunks into a playable/downloadable file
  • CanvasRenderingContext2D — Bar chart rendering from frequency data
