Arquitectura general

Toda la aplicacion es un unico archivo index.html con CSS y JavaScript inline. Zero dependencias externas, zero frameworks, zero bundlers.

El codigo JS se organiza en una IIFE principal ((function () { 'use strict'; ... })();) que contiene IIFEs anidadas, una por demo. Cada una es autonoma: busca su canvas por document.getElementById() y si no existe, retorna silenciosamente.

  (function () {
    'use strict';

    /* Helper compartido para canvas con soporte HiDPI */
    function initCanvas(id, w, h) {
      var cv = document.getElementById(id);
      if (!cv) return null;
      var dpr = window.devicePixelRatio || 1;
      cv.width = w * dpr;
      cv.height = h * dpr;
      var ctx = cv.getContext('2d');
      ctx.scale(dpr, dpr);
      return { cv, ctx, w, h };
    }

    /* Cada demo vive en su propia IIFE */
    (function () {
      /* particulas */ ...
    })();
    (function () {
      /* audio */ ...
    })();
    /* ... 13 demos en total */
  })();

Las 13 demos y que ensenan

Particulas interactivas

Canvas 2D requestAnimationFrame Mouse Events Touch Events

80 particulas que se repelen del cursor y dibujan conexiones entre si cuando estan cerca. Usa requestAnimationFrame para el loop, Math.sqrt para distancias, y getBoundingClientRect() para coordenadas relativas al canvas. Soporta touchmove en dispositivos moviles.

Visualizador de audio

Web Audio API getUserMedia AnalyserNode OscillatorNode

Dos modos: (1) micrófono real via getUserMedia({ audio: true }) conectado a un AnalyserNode que provee datos de frecuencia en un Uint8Array, y (2) sintetizador que genera 4 notas con OscillatorNode + GainNode para el ataque y decaimiento.

Bubble sort vs Quick sort

Async/Await Promises Algoritmos Canvas 2D

Visualizacion en vivo de dos algoritmos de ordenamiento. Usa async/await con setTimeout para controlar la velocidad. Cada paso pausa, redibuja el canvas, y continua. Quick sort implementa particion recursiva con async. El toggle entre algoritmos comparte el mismo arreglo de datos.

Simulacion de fisica

Canvas 2D Fisica clasica

Bolas con gravedad, rebote elastico, friccion y highlights especulares. Por-frame:
b.vy += grav; b.x += b.vx; b.y += b.vy; con deteccion de colision contra bordes del canvas y coeficiente de restitucion variable (e: 0.7 + Math.random() * 0.2).

Fractal de Mandelbrot

Numeros complejos ImageData putImageData

Renderiza el conjunto de Mandelbrot iterando z = z^2 + c hasta 80 iteraciones por pixel. Usa createImageData() para manipular pixeles directamente via Uint8ClampedArray. Click para zoom (reduce viewport 35%). Los colores usan Math.sqrt() y Math.pow() para transiciones suaves.

Dibujo libre

Canvas 2D Mouse Events Touch Events

Paint simple: mousedown inicia el trazo, mousemove dibuja con lineTo() + stroke(), mouseup corta. Integra <input type="color"> para el color y un slider para el grosor. Soporte completo táctil con touchstart/touchmove.

Matrix Rain

Canvas 2D Efecto visual

Columnas de caracteres katakana cayendo con efecto estela. Usa fillStyle = 'rgba(15,15,19,0.06)' sobre el canvas en vez de clearRect para crear el rastro progresivo. Cada columna tiene velocidad aleatoria y se reinicia al llegar al fondo.

Calculadora

DOM Events Logica de estados

Maquina de estados simple: cuatro operaciones, encadenamiento (2 + 3 * 4 = 20), manejo de division por cero, limite de 12 digitos, y formateo con parseFloat().toFixed(10) para evitar errores de punto flotante.

Reloj analogico

Date API Canvas transforms

Tres agujas (hora, minuto, segundo) dibujadas con ctx.save()/ctx.translate()/ctx.rotate() y lineCap: 'round'. La aguja de segundos es suave porque usa getMilliseconds() para interpolacion. Marcas de hora con diferente grosor para las horas principales.

Snake Game

Game loop Keyboard Events Colision detection

Game loop clasico con setTimeout a 120ms. La serpiente se mueve por grid de 12px. Detecta colision contra paredes y contra si misma (snake.some()). Usa e.preventDefault() en keydown para evitar el scroll con flechas. Puntaje y estado de Game Over.

Editor de imagenes

File API Drag & Drop API TypedArrays (Uint8ClampedArray) ImageData

Carga imagenes via FileReader.readAsDataURL() o arrastrando con dragenter/dragover/drop. Aplica filtros manipulando el Uint8ClampedArray del ImageData pixel por pixel. Filtros: grayscale (0.299R+0.587G+0.114B), sepia (matriz 3x3), invert (255 - v), brillo (+40), blur (box blur 5x5).

Web Speech

SpeechRecognition API SpeechSynthesis API

Reconocimiento de voz con SpeechRecognition (prefijo webkit incluido). Escucha en español argentino con resultados intermedios. Síntesis de voz con SpeechSynthesisUtterance configurado con lang: 'es-AR' y rate: 0.9.

Generators + Proxy

function* / yield Proxy / handler Meta-programacion ES6

Generator infinito de Fibonacci con function* fibonacciGen() { yield a; ... }. Cada click avanza al siguiente numero. Un Proxy envuelve un objeto data con handlers get/set que registran cada acceso y se muestran en pantalla. Demostracion de lazy evaluation y meta-programacion.

Fetch API

fetch() Promises (then/catch) JSON parsing

Consume la API JSONPlaceholder con fetch() y encadena .then()/.catch(). Renderiza dinamicamente 10 usuarios con nombre, email y compania. Maneja estados de carga (disabled + texto), error HTTP y error de red con mensajes en pantalla.

Game of Life

Automata celular Canvas 2D Uint8Array

Implementacion del Juego de la Vida de Conway. El tablero se almacena como arreglo de Uint8Array para eficiencia. Cada generacion computa 8 vecinos por celda con bordes toroidales ((r + dr + ROWS) % ROWS). Click para alternar celulas vivas/muertas, botones para pausar, aleatorizar y limpiar.

SEO

La pagina implementa las practicas recomendadas para motores de busqueda:

  • Meta tags — description, keywords, author, viewport
  • Open Graph — og:title, og:description, og:type, og:url para compartir en redes
  • Twitter Cards — twitter:card, twitter:title, twitter:description
  • JSON-LD — Datos estructurados tipo WebApplication con Schema.org
  • Canonical URL — Previene contenido duplicado
  • robots.txt — Permite crawling completo
  • sitemap.xml — Indice de paginas para buscadores
  • HTML semantico — <header>, <main>, <footer>, <article>, <section>

Accesibilidad (a11y)

  • Skip link — <a href="#main" class="skip-link"> para saltar la navegacion
  • Landmarks — Roles ARIA: banner, main, contentinfo
  • Canvas accesibles — role="img" + aria-label descriptivo en cada canvas
  • Estados ARIA — aria-pressed en botones toggle, aria-live="polite" en areas de estado dinamico, role="status"
  • Contraste de color — Texto principal #e4e4ec sobre #0f0f13 (15.9:1), secundario #b0b0c8 (5.8:1), links #a78bfa (7.5:1). WCAG AA en todos los elementos
  • Focus visible — :focus-visible con anillo purpura en botones, inputs y enlaces
  • Reduced motion — @media (prefers-reduced-motion: reduce) desactiva animaciones y transiciones
  • High contrast — @media (prefers-contrast: high) con colores mas fuertes y bordes solidos
  • Teclado — Snake con flechas, botones navegables por tabulacion, labels asociados a inputs

Performance

  • HiDPI / Retina — window.devicePixelRatio escala el buffer interno del canvas para que se vea nitido en pantallas 2x y 3x sin cambiar las coordenadas logicas
  • requestAnimationFrame — Todas las animaciones usan RAF, que sincroniza con el refresco del monitor y pausa en tabs no visibles
  • Zero dependencias — Sin librerias, sin frameworks, sin CDNs. Todo inline, cero requests externos
  • Canvas eficiente — Solo se redibuja lo necesario (clearRect en cada frame), sin acumular memoria
  • Uint8Array — Tipos nativos para datos binarios en Game of Life, analizador de audio y filtros de imagen
  • Touch events — passive: false solo donde se necesita preventDefault, minimizando latencia
